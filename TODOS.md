- **What:** 지원 연도 범위, 윤달 처리, 절입시각 분 단위 경계, 해외 출생 시간대 보정, 서머타임(1948-1988), 양력/음력 입력 UX, 출생지/시간대 필드 추가
- **Why:** 사주 앱의 핵심 정확도. 정의 없이 캐시 키/golden test/유료 스냅샷 구현 불가. Codex 2회 연속 지적
- **Effort:** M (human: 2시간 / CC: 30분) | **Blocks:** L1 엔진 구현, 테스트 전략
- **Priority:** 구현 전 필수

### [P1] Claude API 기반 비용 모델 재산정
- **What:** Ollama Cloud(고정 구독) → Claude Opus 4.6(per-token) 변경에 따른 전체 비용 모델 재계산. per-message 토큰캡, per-session 비용캡, 성능저하 규칙 설계
- **Why:** 기존 마진 97% 계산 전부 무효. Codex 지적: "기존 경제성 모두 사망"
- **Depends on:** Claude API 현재 가격 확인
- **Priority:** 구현 전 필수

### [P2] 유료 결과물 provenance 모델 설계
- **What:** 각 유료 결과에 엔진 버전, 슬롯 스키마 버전, 프롬프트 버전, 모델 버전 기록. 결과 재현 및 분쟁 해결 가능
- **Why:** Codex 지적 — provenance 없으면 결과 재현, 분쟁 설명, 엔진 변경 안전성 보장 불가
- **Depends on:** 슬롯 스키마 버전닝 설계
- **Priority:** Phase 1 구현 시

### [P2] 유료 결과물 불변성(immutability) 정책 정의
- **What:** 사용자가 출생시간 추가/프로필 변경 시, 또는 엔진 수정 시 기존 유료 결과의 처리 방침 명시
- **Why:** Codex 지적 — 정하지 않으면 이력이 신뢰불가
- **Priority:** 설계 단계

### [P2] 아티팩트 수준 상태 관리

- **Depends on:** Worker 아키텍처
- **Priority:** 구현 시
