## Problem Statement

한국 사주/운세 시장은 약 1.4조원 규모이며, MZ세대 중심으로 AI 사주 앱이 급성장 중이다. 현재 시장은 **무료 대량 서비스(점신, 포스텔러)**와 **프리미엄 일방향 콘텐츠(청월당)**로 양극화되어 있다.

청월당(3-5만원)은 웹툰 형식의 고품질 결과물을 제공하지만, 결과를 보여주고 끝나는 일방향 구조이다. 사용자가 "내 사주에서 올해 이직해도 될까?"같은 후속 질문을 하고 싶어도 추가 결제가 필요하다.

**기회**: AI 채팅으로 양방향 대화가 가능한 사주 상담 + NanoBanana API로 Lottie풍(귀엽고 플랫한 벡터 일러스트) AI 이미지/GIF 결과물 + 앱 내 카드 공유로 유입을 결합한 모바일 앱 서비스.

## Demand Evidence

- 한국 점술 시장 약 1.4조원, 10~39세 1인당 약 8만원 소비
- 점신(테크랩스): 매출 830억원, 1,500만 다운로드
- 포스텔러(운칠기삼): 매출 100억원, 860만 가입자
- 청월당(로켓AI): ChatGPT 스토어 라이프스타일 세계 1위
- 창업자 주변에 사주 앱에 정기적으로 결제하는 사용자 존재 (구체적 인원/금액 미확인)
- **수요 검증 상태: 약함** — 시장 규모는 크지만 개인적 수요 검증은 아직 이루어지지 않음

## Status Quo

현재 사주/운세를 보고 싶은 사람들의 선택지:

| 서비스 | 가격 | 특징 | 한계 |
|--------|------|------|------|
| 청월당 | 3-5만원 | 프리미엄 웹툰 사주 | 일방향 결과물, 비쌈 |
| 점신 | 무료~소액 | 1,500만 DL, 다양한 운세 | 결과 품질 낮음 |
| 포스텔러 | 무료~소액 | 점성술+사주+타로 | 과금 유도 심함 |
| 오프라인 사주카페 | 3-10만원 | 대면 상담, 신뢰감 | 비싸고 접근성 낮음 |
| 유튜브 사주 영상 | 무료 | 일반적 운세 | 개인화 없음 |

**빈틈**: 대화형으로 후속 질문이 가능하면서도 시각적으로 매력적인 결과물을 제공하는 서비스가 없다.

## Target User & Narrowest Wedge

**1차 타겟**: MZ세대 전체 (20~30대, 남녀 무관)
- 사주/운세에 관심이 있지만 3-5만원은 부담스러운 층
- MBTI, 타로 등 자기분석 콘텐츠를 SNS에 공유하는 습관이 있는 층
- 기존 무료 앱의 결과가 너무 단순하다고 느끼는 층

**첨언**: 타겟이 넓다. 런칭 후 데이터를 보고 빠르게 좁혀야 한다. 예상 핵심 세그먼트: 20대 중후반 여성, 연애/결혼 고민.

**Narrowest Wedge**: 사주 카드 (무료, NanoBanana Lottie풍 이미지) → 이미지 공유(카카오톡/인스타) → 앱 설치 유도 → 유료 AI 사주 상담 전환

## Constraints

- 1인 또는 소규모 팀으로 시작
- 사주 해석 품질: 만세력 기반 정확한 사주팔자 계산 필요 (3-layer 엔진이 핵심 차별화)
- 법적: 통신판매업 신고, 개인정보처리방침 필수
- 사주 콘텐츠 전문성: 명리학 전문가 자문 또는 검증 필요
- IAP 수수료: Apple/Google 30% 수수료가 마진에 미치는 영향 관리

## Premises

1. **가격 1-1.5만원은 진입장벽을 낮추는 수단**이지 핵심 가치가 아니다 ✅ (단, IAP 30% 수수료 반영 필요)
2. **NanoBanana API로 생성한 Lottie풍 AI 이미지/GIF가 차별화**를 만든다 ✅ (귀엽고 플랫한 벡터 일러스트 스타일, 사용자가 공유하고 싶은 매력적인 비주얼)
3. **콘텐츠의 재미와 품질이 구매 결정의 핵심**이다 ✅ (3-layer 사주 엔진이 핵심 경쟁력 — 청월당의 "모순 분석" 문제 해결)
4. **대화형 AI 상담이 일방향 결과물보다 높은 만족도**를 줄 것이다 ⚠️ (미검증, 합리적)
5. **앱 내 사주 카드 이미지 공유가 유입에 기여**할 것이다 ⚠️ (모바일 앱 특성상 URL 공유 불가, 이미지+딥링크 공유 → 바이럴 효과 검증 필요)

## Approaches Considered

### Approach A: 웹 퍼스트 (Next.js + FLUX AI 이미지)
- Next.js + Vercel + FLUX 이미지 생성 + GIF 파이프라인
- Effort: L | Risk: High
- **기각 사유**: GIF 파이프라인(BullMQ + FFmpeg) 복잡성, AI 이미지 비용(200원/장, 바이럴 시 월 2억), Vercel 서버리스 타임아웃 제약

### Approach B: 모바일 퍼스트 (Flutter + Rust) ← 채택
- Flutter + Rust (Axum) + NanoBanana API (Lottie풍 AI 이미지/GIF 생성)
- 차별화: 3-layer 사주 엔진 정확성 + 양방향 AI 채팅
- Effort: M-L | Risk: Med

### Approach C: 하이브리드 (웹 MVP → 모바일)
- Next.js 웹 MVP 후 Flutter 추가
- Effort: M | Risk: Low
- **기각 사유**: 사용자가 모바일 퍼스트를 선택

## 채택: Approach B — 모바일 퍼스트 (Flutter + Rust)

### 서비스 구조

```
[무료] 사주 카드 (NanoBanana Lottie풍 이미지)
  ├── 생년월일 입력 (로그인 불필요)
  ├── Rust 엔진이 사주 계산 → NanoBanana API로 Lottie풍 카드 이미지 생성
  ├── 카드 이미지 저장 → 공유 시트 (카카오톡, 인스타, 시스템 공유)
  └── "더 자세히 보기" → 유료 전환 유도

[유료] AI 사주 상담 (15,000원, IAP)
  ├── Step 1: 기본 정보 입력 (생년월일, 성별, 출생시간)
  ├── Step 2: 상담 주제 선택 (연애, 재물, 커리어, 건강, 종합)
  ├── Step 3: IAP 결제 (RevenueCat)
  ├── Step 4: 서버에서 L1+L2 계산 → L3 LLM 해석 생성
  ├── Step 5: NanoBanana로 Lottie풍 결과 이미지/GIF 시퀀스 생성 (5-8장)
  ├── Step 6: AI 사주 캐릭터 "월하선생"과 채팅 상담
  │   ├── "올해 이직해도 될까요?"
  │   ├── "애인과 궁합이 어때요?"
  │   └── 세션 내 질문 가능 (72시간, 최대 50턴)
  └── Step 7: 결과 저장 + 이미지 공유

[무료] 궁합 미리보기 (Phase 1)
  ├── 두 사람 생년월일 입력
  ├── L1+L2 계산으로 궁합 점수 + 한줄 요약 (무료)
  └── "상세 궁합 분석 보기" → 유료 전환 유도

[유료] 궁합 상담 (12,000원, IAP)
  ├── 상세 궁합 분석 + NanoBanana 이미지 4-6장
  └── AI 채팅 (72시간, 50턴)

[무료] 오늘의 운세 (Phase 1, 리텐션 훅)
  ├── 생년월일 기반 일간 운세 텍스트 (Claude API, 이미지 생성 없음)
  ├── 매일 접속 유도 + 푸시 알림
  └── iOS 위젯 지원 (홈 화면에서 바로 확인)

[무료] 평생 사주 프로필 (Phase 1)
  ├── 최초 1회 사주 분석 후 영구 프로필 생성
  ├── 오행 차트 시각화 (木火土金水 밸런스 그래프)
  └── 프로필 기반으로 오늘의 운세/유료 상담 개인화

[Phase 1] 캐릭터 선택
  ├── AI 상담 캐릭터 선택 가능 (v1: "월하선생" 1종)
  └── 향후 캐릭터 추가 (여성형, 친구형 등)

[v2] 꿈해몽 AI, 이름 풀이, 구독 모델, 웹 버전
```

**핵심 포인트: 결제는 LLM 해석(비용 발생) 이전에 수행**. L1+L2는 Rust 로컬 계산이라 비용 0 — 사전 미리보기(오행 요약 등)로 구매 동기 부여 가능.

### 핵심 기능 상세

**1. 사주 카드 (유입 엔진) — 무료**
- 생년월일만 입력하면 Rust 엔진이 사주 계산 → NanoBanana API로 Lottie풍 카드 이미지 생성
- **카드 내용**: 일주 요약, 올해 키워드 3개, 행운 요소 1개 (티저 수준)
- **이미지 스타일**: Lottie풍 — 귀엽고 플랫한 벡터 일러스트, 파스텔 톤, 동양적 모티프 (오행 색상 활용)
- **포맷**: NanoBanana 이미지 (PNG), 공유 시 그대로 전달
- 공유: 카카오톡 공유 API + 인스타 스토리 + 시스템 공유 시트 (딥링크 포함 → 앱 미설치 시 스토어로 이동)
- 비용: **~27원/장** (NanoBanana API ~$0.02/image). 동일 생년월일은 캐싱하여 재생성 비용 0원.
- **Rate Limit**: 기기당 1일 3회. 초과 시 로그인 요구.
- **생성 시간**: <10초 (NanoBanana). "당신의 사주를 그리고 있습니다..." 로딩 표시.

**2. AI 사주 상담 (핵심 수익원) — 유료 (IAP 15,000원)**
- 만세력 기반 3-layer 사주 엔진으로 정확한 계산
- **AI 사주 캐릭터 페르소나 (v1)**:
  - 이름: "월하선생" (남성형 현자 캐릭터)
  - 말투: 존댓말, 따뜻하지만 직설적. "~입니다", "~하시겠습니다"
  - 톤: 전통 명리학자가 현대어로 설명하는 느낌
  - 예시: "당신의 일주는 갑목(甲木)이군요. 큰 나무처럼 곧은 성품을 지니셨습니다. 올해 병오년은 나무에서 열매가 무르익는 해입니다 — 그간의 노력이 결실을 맺겠습니다."
- **유료 결과물**: NanoBanana Lottie풍 이미지/GIF 시퀀스 (주제별 상세 분석, 5-8장)
- 결과 확인 후 AI 캐릭터와 채팅으로 후속 질문 가능
- **채팅 세션 정의**: 결제일로부터 72시간 유효, 최대 50턴(질문+답변 각 1턴), 세션 만료 후 결과물은 영구 열람 가능하나 추가 채팅 불가
- **채팅 컨텍스트 전략**: 사주 분석 요약본(~2,000 tokens)을 system prompt에 포함, 대화 히스토리는 최근 20턴 유지, 오래된 턴은 요약하여 압축 (요약 로직은 Claude API로 처리 — per-token 비용 발생, 토큰캡으로 관리)

**3. 궁합 미리보기 — 무료 (Phase 1)**
- 두 사람의 생년월일 입력 → L1+L2로 궁합 점수 + 한줄 요약 무료 제공
- **무료 내용**: 궁합 점수(100점 만점), 성격 궁합 한줄 요약
- **유료 전환**: "상세 궁합 분석 + AI 상담" → IAP 12,000원
- 비용: 0원 (Rust L1+L2만 사용, LLM/이미지 호출 없음)

**4. 오늘의 운세 — 무료 (Phase 1, 리텐션 훅)**
- 생년월일 기반 일간 운세 텍스트 (Claude API, 이미지 생성 없음 — 비용 최소화)
- 매일 접속 유도 + 푸시 알림 (FCM)
- **iOS 위젯**: 홈 화면에서 바로 오늘의 운세 확인 가능 → 앱 재방문율 극대화
- 비용: Claude API per-token 비용 (토큰캡으로 관리)

**5. 평생 사주 프로필 — 무료 (Phase 1)**
- 최초 사주 분석 시 영구 프로필 자동 생성
- **오행 차트**: 木火土金水 밸런스를 시각적 그래프로 표시 (앱 내 렌더링, NanoBanana 불필요)
- 프로필 데이터를 기반으로 오늘의 운세, 유료 상담 결과 개인화
- 로그인 필수 (프로필 저장을 위해)

**6. 캐릭터 선택 (Phase 1 기반, 향후 확장)**
- v1: "월하선생" 1종 (남성형 현자)
- 향후: 여성형 캐릭터, 친구형 캐릭터 등 추가 → 캐릭터별 말투/톤 차별화
- 캐릭터 선택은 채팅 시작 전 1회 선택

**무료 vs 유료 비교:**

| 항목 | 무료 사주 카드 | 유료 사주 상담 |
|------|--------------|--------------|
| 내용 깊이 | 키워드 3개 + 요약 1줄 | 주제별 상세 분석 (5-8 화면) |
| AI 채팅 | 불가 | 72시간 50턴 |
| 이미지 | NanoBanana Lottie풍 1장 | NanoBanana Lottie풍 5-8장 (GIF 시퀀스) |
| 비용 (서버) | ~27원 (이미지 1장) | ~200원 (이미지 5-8장, LLM은 구독 고정비) |
| 용도 | 앱 유입 (티저) | 깊이 있는 상담 |

### 생년월일 입력 UI (/plan-design-review 신규)

사주 앱의 가장 핵심 입력. 모든 기능(무료 카드, 유료 상담, 궁합)에서 재사용되는 공통 컴포넌트.

```

│  음력 선택 + 유효하지 않은 날짜   │

│  않습니다" 인라인 에러            │
├─────────────────────────────────┤
│  성별                           │
│  [남 ●] [여 ○]                 │ ← 세그먼트 토글
├─────────────────────────────────┤
│  출생시간                        │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ │
```

**입력 규칙:**
- 양력 기본, 음력 선택 시 윤달 체크박스 노출
- 음력 날짜 유효성 검증: 변환 테이블에 없는 날짜는 인라인 에러
- 시진: 야자시(23:00~01:00) 처리 — 야자시론 기본, 조자시론은 설정에서 변경 가능
- 서머타임 보정: 1948-1988 한국 적용 시 자동 안내 표시
- "모름" 선택 시: 시주 제외 계산. 결과에 영구 배너 "출생시간을 추가하면 더 정확한 분석 가능"

### Happy Path (유료 사주 상담)

```
사용자 앱 실행
  → 소셜 로그인 (카카오 / Apple / Google) → JWT 발급
  → 생년월일 + 성별 + 출생시간 입력
  → Rust L1 만세력 계산 + L2 검증 (비용 0원, 서버 < 50ms)
  → 오행 요약 미리보기 표시 ("당신은 갑목 일주입니다")
  → 상담 주제 선택 (연애/재물/커리어/건강/종합)
  → IAP 결제 (RevenueCat, 15,000원)
  → 서버 receipt 검증
  → L3 LLM 해석 생성 (Claude API, 스트리밍, 3-8초)
  → NanoBanana로 Lottie풍 결과 이미지/GIF 생성 (5-8장, ~30-60초)
  → 결과 저장 (DB) + 이미지 시퀀스 표시
  → AI 채팅 세션 시작 ("월하선생"과 대화)
```

### Failure States & 예외 처리

| 실패 상황 | 대응 |
|----------|------|
| IAP 결제 취소 | 입력 화면으로 복귀. "결제가 취소되었습니다" |
| IAP 결제 실패 (카드 오류 등) | "결제에 실패했습니다. 다시 시도해주세요" + 재시도 |
| IAP 결제 성공 + 서버 장애 | RevenueCat webhook 자동 재시도. 복구 후 결과 생성. 사용자에게 푸시 알림 |
| IAP 결제 성공 + LLM 해석 실패 | Retry 2x → 실패 시 L1+L2 데이터 기반 폴백 텍스트 + "상세 해석이 준비되면 알림을 보내드립니다" |
| IAP 환불 | Apple/Google 환불 정책에 따름 (개발자가 직접 환불 불가, RevenueCat 이벤트로 감지 → 세션 비활성화) |
| AI 채팅 API 다운 | "잠시 후 다시 시도해주세요" 안내 + 세션 유효기간 연장(다운타임만큼) |
| 무료 카드 이미지 생성 실패 | Retry 2x → 실패 시 텍스트 기반 폴백 카드 표시 + "이미지가 준비되면 알림" |
| NanoBanana API 다운 | 텍스트 기반 사주 결과 먼저 표시 + "이미지가 준비되면 알림을 보내드립니다" |
| 채팅이 사주 범위 이탈 | System prompt 경계 + "사주와 관련된 질문에 답변드리고 있습니다" |
| 출생시간 모름 | "출생시간 모름" 선택 → 시주 제외 계산. "출생시간을 아시면 더 정확한 분석 가능" CTA |

### 사주 연산 엔진 (Rust 네이티브, 3-Layer)

청월당의 가장 큰 약점이 **"목 과다" vs "목 부족" 같은 분석 모순**이었음.
이를 방지하기 위해 **연산 계층과 해석 계층을 철저히 분리**.

```
┌─────────────────────────────────────────┐
│  Layer 1: 만세력 연산 (Rust, 결정론적)      │
│  - 양력→음력 변환 (자체 변환 테이블 구현)    │
│  - 24절기 절입시각 (한국천문연구원 데이터)    │
│  - 천간/지지 계산 (연주, 월주, 일주, 시주)   │
│  - 오행 배분 및 점수화                      │
│  - 십신 관계 매핑                          │
│  - 대운/세운/월운 계산                      │
│  - 신살 판별                              │
│  출력: 구조화된 JSON (숫자, enum 값)        │
│                                           │
│  ⚠️ 주의 구현 사항:                        │
│  - 야자시(23:00~01:00) 처리: 야자시론 기본   │
│    + 조자시론 옵션 (사용자 선택 가능)        │
│  - 절기 기반 월주 계산: 음력 월이 아님       │
│  - 대운 시작 나이: 남녀 순행/역행 구분       │
│    (생일~절기 일수÷3)                      │
│  - 서머타임 보정 (1948-1988 한국 적용)       │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│  Layer 2: 검증 계층 (Rust, 규칙 기반)       │
│  - 오행 과다/부족 판정을 수치로 확정          │
│  - 모순 탐지: 같은 오행에 상반된 판정 불가     │
│  - 결과 일관성 보장                         │
│  출력: 검증된 분석 데이터                    │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│  Layer 3: 해석 생성 (Anthropic Claude API)    │
│  - 검증된 데이터를 프롬프트에 주입            │
│  - "목 점수 8/10 (과다)" 같은 확정값 전달    │
│  - LLM은 수치를 자연어로 변환만 담당          │
│  - 생성 후 재검증: 원본 데이터와 불일치 탐지   │
│  - 프롬프트: prompts/ 디렉토리에 외부 관리    │
│    (하드코딩 금지, A/B 테스트 가능)          │
│  출력: 사용자용 해석 텍스트                  │
│  ⚠️ 비용 발생 구간: 유료 결제 후에만 실행     │
└─────────────────────────────────────────┘
```

**핵심 설계 원칙:**
- Layer 1은 **결정론적** — 같은 입력에 항상 같은 결과 (테스트 가능, golden file 50+건)
- Layer 2가 **모순 방지 게이트** — 청월당이 비판받은 핵심 문제 해결
- Layer 3의 LLM은 **"번역기"** 역할만 — 분석 판단을 LLM에 위임하지 않음
- **음력 변환**: `chrono` 크레이트는 날짜/시간 기본 연산만 담당. 음력↔양력 변환은 **자체 변환 테이블** 구현 필요 (chrono에 음력 기능 없음)

### 앱 화면 구조 (모바일)

```
Tab Bar (4탭):
├── 홈 (사주 카드 + 오늘의 운세 + 유료 상담/궁합 CTA)
├── 내 프로필 (평생 사주 프로필 + 오행 차트)
├── 내 분석 (히스토리 + 진행 중 채팅)
└── 설정 (계정, 캐릭터 선택, 알림, 고객지원, 약관)

Navigation Stack:
홈 → 무료 카드 생성 (입력 → NanoBanana 이미지 → 공유)
홈 → 오늘의 운세 (텍스트, 매일 갱신)
홈 → 궁합 미리보기 (2인 입력 → 무료 점수 → 유료 전환)
홈 → 유료 상담 (입력 → 미리보기 → 주제 선택 → 결제 → 결과 → 채팅)
내 프로필 → 오행 차트 + 사주 요약 + 성향 분석
내 분석 → 과거 결과 상세 → 채팅 이력 (읽기 전용 or 활성)
설정 → 캐릭터 선택, 계정 관리, 알림 설정, 데이터 삭제 요청, 약관
```
