## 1. 사전 준비 (공통)

### 1.1 법인/사업자 등록

- [ ] **사업자등록** (개인사업자 또는 법인)
  - 관할 세무서에서 발급 (온라인: 홈택스)
  - 업태: 정보통신업 / 업종: 소프트웨어 개발 및 공급업, 응용소프트웨어 개발 및 공급업
  - 소요 기간: 즉일~3일
  - 비용: 무료

- [ ] **통신판매업 신고** (인앱결제가 있으므로 필수)
  - 관할 시/군/구청 또는 정부24 온라인 신고
  - 사업자등록증 필요
  - 소요 기간: 3~5 영업일
  - 비용: 무료 (일부 지자체 수수료 있을 수 있음)
  - **주의**: 통신판매업 신고번호를 앱/웹에 표시해야 함

### 1.2 필수 법적 문서 작성

- [ ] **개인정보처리방침 (Privacy Policy)**
  - 한국어 + 영문 버전 필요
  - 호스팅: 웹사이트에 게시 (예: `https://saju.app/privacy`)
  - 필수 포함 항목 (개인정보보호법/PIPA 기준):
    - 수집하는 개인정보 항목 (생년월일, 출생시간, 이메일, 소셜 로그인 정보)
    - 수집 및 이용 목적
    - 보유 및 이용 기간
    - 제3자 제공 (Claude API로 AI 처리 시 해외 이전 고지 필수)
    - 개인정보 처리 위탁 (Supabase, RevenueCat, Firebase 등)
    - 개인정보의 국외 이전 (Anthropic - 미국, Google Firebase - 미국 등)
    - 개인정보보호책임자 연락처
    - 정보주체의 권리와 행사 방법
    - 쿠키 사용 여부
    - 개인정보 파기 절차
    - 14세 미만 아동 관련 사항

- [ ] **이용약관 (Terms of Service)**
  - 한국어 + 영문 버전
  - 호스팅: `https://saju.app/terms`
  - 필수 포함:
    - 서비스 설명 (AI 기반 엔터테인먼트 목적, 전문 상담 아님 명시)
    - 결제/환불 정책
    - 면책 조항 (사주 결과는 오락 목적이며 실제 운명을 예측하지 않음)
    - 계정 탈퇴 방법
    - 분쟁 해결 (한국 법원 관할)
    - 지적재산권

- [ ] **환불 정책**
  - 앱스토어/플레이스토어 기본 환불 정책 준수
  - 디지털 콘텐츠 특성상 소비 후 환불 불가 명시 (전자상거래법 준수)

### 1.3 필수 기능 구현

- [ ] **계정 삭제 기능** (Apple/Google 모두 필수)
  - 앱 내에서 직접 계정 삭제 가능해야 함
  - 삭제 시 모든 사용자 데이터 완전 삭제 (또는 삭제 일정 안내)
  - 삭제 확인 이메일/알림

- [ ] **로그인 관련**
  - Apple 로그인 필수 (iOS에서 소셜 로그인 제공 시 Apple Sign In 반드시 포함)
  - Kakao/Google 로그인

- [ ] **엔터테인먼트 면책 고지**
  - 앱 내 명확한 고지: "본 서비스는 엔터테인먼트 목적이며, 전문적인 상담을 대체하지 않습니다"
  - 상담 결과 화면에 면책 문구 표시

### 1.4 앱 아이콘 및 에셋 준비

- [ ] **앱 아이콘**
  - iOS: 1024x1024px (App Store용) — 알파 채널 없이, 모서리 자르기 없이
  - Android: 512x512px (Play Store용)
  - Adaptive Icon (Android): 108x108dp foreground + background layer

## 2. 한국 법적 요구사항

### 2.1 개인정보보호법 (PIPA) 준수

- [ ] **개인정보 수집 동의**: 회원가입 시 명시적 동의 UI (체크박스, 선택적/필수 분리)
  - 필수: 서비스 이용에 필요한 최소한의 정보
  - 선택: 마케팅 수신 동의 등
- [ ] **개인정보 처리방침 공개**: 앱 내 + 웹사이트
- [ ] **개인정보 암호화**: 생년월일/출생시간 암호화 저장 (이미 구현됨 - crypto.rs)
- [ ] **개인정보 국외이전 동의**: Claude API (Anthropic, 미국) 이용 시 별도 동의
  - 이전 국가, 이전 일시 및 방법, 이전 목적, 수령자 정보 고지
- [ ] **개인정보영향평가**: 5만 명 이상 민감정보 처리 시 필요 (MVP 단계에서는 해당 없을 가능성)

### 2.2 정보통신망법 준수

- [ ] **광고성 정보 전송 시 수신 동의**: 푸시 알림으로 마케팅 시 사전 동의 필요
- [ ] **수신 거부 수단 제공**: 푸시 알림 설정에서 끌 수 있도록
- [ ] **광고 표시**: 광고성 메시지에 "(광고)" 표시 의무

### 2.3 전자상거래법

- [ ] **사업자 정보 표시** (앱 내 또는 웹):
  - 상호, 대표자명, 사업장 주소, 전화번호
  - 사업자등록번호
  - 통신판매업 신고번호
  - 이메일
- [ ] **청약 철회 안내**: 디지털 콘텐츠 특성상 이용 후 철회 불가 고지

### 2.4 사주/운세 앱 특수 사항

- [ ] **전문 상담 면책**: "본 서비스는 오락 목적이며, 의학/법률/심리 상담을 대체하지 않습니다"
- [ ] **AI 생성 콘텐츠 표시**: AI가 생성한 결과임을 명시
  - 예: "이 상담 결과는 AI에 의해 생성되었습니다"
- [ ] 사주/운세 서비스 자체는 한국에서 별도 허가 불요 (엔터테인먼트로 분류)
- [ ] 단, 대면 상담/점술업은 별도 (앱은 해당 없음)

## 3. Apple App Store (iOS)

### 3.1 Apple Developer Program 등록

- [ ] Apple ID 준비 (2FA 필수)
- [ ] **Apple Developer Program 가입**
  - 개인: 연간 $99 (약 13만원) — 본인 확인 필요
  - 법인: 연간 $99 — D-U-N-S 번호 필요 (발급 2~4주 소요)
  - **권장: 개인 개발자로 시작** (MVP 단계)
- [ ] 가입 승인 대기 (보통 24~48시간, 경우에 따라 더 걸림)

### 3.2 App Store Connect 설정

- [ ] **앱 등록** (App Store Connect → 내 앱 → 새 앱)
  - 번들 ID: `com.saju.app` (또는 본인 도메인 역순)
  - SKU: 내부 관리용 고유 ID
  - 기본 언어: 한국어

- [ ] **앱 정보 입력**
  - 앱 이름: 30자 이내 (예: "사주 - AI 사주 상담")
  - 부제: 30자 이내 (예: "사주팔자로 보는 나의 운명")
  - 카테고리: 주 카테고리 "라이프스타일" 또는 "엔터테인먼트"
  - 보조 카테고리: "엔터테인먼트" 또는 "참고"

### 3.3 필수 에셋

- [ ] **스크린샷** (최소 요구사항)
  - 6.7인치 (iPhone 15 Pro Max): 1290 x 2796px — 최소 2장, 최대 10장
  - 6.5인치 (iPhone 14 Plus): 1284 x 2778px — 최소 2장
  - 5.5인치 (iPhone 8 Plus): 1242 x 2208px — 선택사항이지만 권장
  - iPad Pro 12.9인치: 2048 x 2732px — iPad 지원 시 필수
  - **팁**: 6.7인치용으로 만든 후 리사이즈 가능 (미디어 관리자에서 하나로 통합 가능)

- [ ] **앱 미리보기 영상** (선택, 강력 권장)
  - 15~30초, 앱 실제 동작 녹화
  - 해상도: 스크린샷과 동일

- [ ] **앱 아이콘**: 1024x1024px PNG (투명 배경 불가, 둥근 모서리 자동 처리)

### 3.4 앱 설명 및 메타데이터

- [ ] **설명문** (최대 4000자)
  - 첫 3줄이 가장 중요 (접기 전에 보이는 영역)

- [ ] **키워드** (최대 100자, 쉼표 구분)
  - 예: 사주,팔자,운세,궁합,AI상담,오늘의운세,사주팔자,토정비결,신년운세

- [ ] **프로모션 텍스트** (최대 170자, 심사 없이 수시 변경 가능)

- [ ] **지원 URL**: `https://saju.app/support`
- [ ] **마케팅 URL**: `https://saju.app` (선택)
- [ ] **개인정보 처리방침 URL**: `https://saju.app/privacy` (필수)

### 3.5 App Review Guidelines 주요 체크포인트

#### 3.5.1 콘텐츠 관련 (Guideline 1.x)

- [ ] **1.1 부적절한 콘텐츠**: 사주 결과에 폭력적/성적/차별적 내용 생성 방지
  - AI 프롬프트에 안전 가드레일 설정
  - 부정적 결과도 건설적으로 표현되도록 프롬프트 엔지니어링

- [ ] **1.2 사용자 생성 콘텐츠**: 해당 없음 (AI 생성이므로)

#### 3.5.2 기능 관련 (Guideline 2.x)

- [ ] **2.1 앱 완성도**: 크래시 없이 안정적으로 동작
- [ ] **2.3 정확한 메타데이터**: 스크린샷/설명이 실제 앱과 일치
- [ ] **2.5.11 SiriKit/Shortcuts**: 해당 없음

#### 3.5.3 비즈니스 관련 (Guideline 3.x)

- [ ] **3.1.1 인앱결제**: 디지털 콘텐츠/서비스는 반드시 Apple IAP 사용
  - RevenueCat을 통한 Apple IAP 연동 필수
  - 외부 결제 유도 금지 (Super App 예외 아닌 이상)

- [ ] **3.1.2 구독**:
  - 구독 상품 설명에 가격, 기간, 자동 갱신 여부 명시
  - 무료 체험 기간 명시 (있는 경우)
  - 구독 관리/해지 방법 안내

- [ ] **3.1.3(b) 멀티플랫폼 서비스**: 안드로이드에서 구매한 크레딧을 iOS에서 사용 불가 (각 플랫폼 IAP 분리)

#### 3.5.4 디자인 관련 (Guideline 4.x)

- [ ] **4.0 디자인 (일반)**: 단순 웹뷰 래핑이 아닌 네이티브 경험 필요
- [ ] **4.2 최소 기능**: "웹사이트를 앱으로 감싼 것" 리젝 가능
- [ ] **4.7 HTML5 게임/앱**: 해당 없음

#### 3.5.5 법적 관련 (Guideline 5.x)

- [ ] **5.1 개인정보**
  - **5.1.1 데이터 수집/저장**: 목적에 필요한 최소 데이터만 수집
  - **5.1.2 데이터 이용/공유**: 동의 없이 제3자 공유 금지
  - **5.1.4 Kids Category**: 14세 미만 대상 아닌 경우 Kids Category 아님

- [ ] **5.1.1(vii) 계정 삭제**: 앱 내에서 계정 삭제 기능 필수
  - Apple Sign In으로 생성한 계정은 Apple의 REST API로 토큰 해제도 필요

### 3.6 Privacy (앱 추적 투명성 & 개인정보 라벨)

- [ ] **App Tracking Transparency (ATT)**
  - IDFA 사용 시 반드시 ATT 팝업 표시 (iOS 14.5+)
  - Firebase Analytics만 사용하고 IDFA 미수집 시 ATT 불필요
  - `Info.plist`에 `NSUserTrackingUsageDescription` 추가 (ATT 사용 시)

- [ ] **Privacy Nutrition Labels** (App Store Connect에서 설정)
  - 수집하는 데이터 유형별로 체크:
    - **연락처 정보**: 이메일 주소 (소셜 로그인 시)
    - **식별자**: 사용자 ID
    - **사용 데이터**: 제품 상호작용
    - **구매**: 구매 내역
    - **기타 데이터**: 생년월일, 출생시간 (사주 계산용)
  - 각 항목에 대해:
    - 수집 여부
    - 사용자 추적에 사용 여부
    - 사용자 연결 여부
  - **주의**: 제3자 SDK도 포함해야 함 (Firebase, RevenueCat 등)

### 3.7 인앱결제 상품 등록

- [ ] App Store Connect → 인앱 구매 → 상품 등록
  - 소모품 (Consumable): 1회 상담권
  - 비소모품 (Non-consumable): 해당 없을 가능성
  - 자동 갱신 구독 (Auto-renewable): 월간/연간 플랜 (있는 경우)
- [ ] 각 상품에 대해:
  - 제품 ID (예: `com.saju.app.consultation_single`)
  - 참조 이름
  - 가격 (Apple 가격 등급 선택)
  - 현지화된 표시 이름 및 설명
  - 심사용 스크린샷
- [ ] **샌드박스 테스트 계정** 생성 및 테스트 완료

### 3.8 빌드 제출

- [ ] Xcode에서 Archive → App Store Connect 업로드
- [ ] 또는 CI/CD (GitHub Actions + fastlane) 사용
- [ ] **빌드 설정 체크**:
  - 최소 iOS 버전: 16.0 권장 (또는 15.0)
  - arm64 아키텍처
  - Bitcode: 더 이상 필요 없음 (Xcode 14+)
  - Swift 패키지/CocoaPods 의존성 모두 포함

### 3.9 심사 제출

- [ ] **심사 노트** 작성 (앱 심사 팀에게 전달):
  - 테스트 계정 정보 (소셜 로그인 우회용 데모 계정 또는 설명)
  - 인앱결제 테스트 방법
  - 앱의 주요 기능 설명
  - AI 사용에 대한 설명

- [ ] **연락처 정보**: 심사 중 연락 가능한 전화번호/이메일

- [ ] **수출 규정 준수**:
  - 암호화 사용 여부 (HTTPS만 사용 시 "예, 면제 대상")
  - `Info.plist`에 `ITSAppUsesNonExemptEncryption` = NO (표준 HTTPS만 사용 시)

- [ ] **연령 등급 설정** (자체 설문):
  - 빈도가 낮거나 경미한 성인/선정적 주제: "드물게/경미하게" 선택 가능
  - 도박 시뮬레이션: 없음
  - 의료/의학 정보: 없음
  - 결과: 아마도 **12+** 또는 **17+** (사주가 미신/점술 관련이므로 보수적으로)

## 5. 인앱결제 (RevenueCat)

### 5.1 RevenueCat 설정

- [ ] RevenueCat 계정 생성 (무료 플랜: 월 $2,500 수익까지 무료)
- [ ] 프로젝트 생성

### 5.2 Apple 연동

- [ ] App Store Connect에서 **Shared Secret** 생성 → RevenueCat에 입력
- [ ] App Store Connect에서 **App-Specific Shared Secret** 또는 **Account Shared Secret** 사용
- [ ] **App Store Server Notifications V2** URL 설정
  - RevenueCat 대시보드에서 URL 복사 → App Store Connect에 입력
- [ ] IAP 상품을 RevenueCat의 **Products**에 매핑

### 5.3 Google 연동

- [ ] Google Play Console에서 **서비스 계정** 생성
  - Google Cloud Console → IAM → 서비스 계정
  - 역할: "재무 데이터 보기, 주문 관리" 권한
  - JSON 키 다운로드 → RevenueCat에 업로드
- [ ] Google Play Console → API 액세스에서 서비스 계정 연결
- [ ] **실시간 개발자 알림 (RTDN)** 설정
  - Google Cloud Pub/Sub 주제 생성
  - RevenueCat에서 제공하는 구독 주소 연결
- [ ] IAP 상품을 RevenueCat의 **Products**에 매핑

### 5.4 RevenueCat 공통

- [ ] **Entitlements** 설정 (예: "premium", "consultation")
- [ ] **Offerings** 설정 (사용자에게 보여줄 상품 패키지)
- [ ] **Paywall** 구성 (앱 내 결제 화면)
- [ ] Flutter SDK 연동 (`purchases_flutter: ^6.17.0` — 이미 추가됨)
- [ ] 테스트:
  - iOS Sandbox 환경 테스트
  - Google Play 라이선스 테스터 환경 테스트
  - 구매 → 복원 → 취소 플로우 전체 테스트
